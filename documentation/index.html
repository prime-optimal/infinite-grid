A) IMPORTANT NOTES - make sure you read this! - top

Please note that the IGP installation and configuration is not complicated but because it has a lot of customizable settings it might seem complicated, please go through the entire documentation before trying to install it into your own page. Basically what it must be done is to copy some html and js code from the examples we provided and paste it into your own html page and of course add your own content.

The server is character case sensitive so make sure that the IGP settings are identical to those from the provided examples.

IGP requires a minimum of 9 thumbnails in each category.

B) Preparing thumbnails and images! - top

Coding this grid was a challenge, first because the logic behind the code is complex and second and the most difficult part is that IGP is pushing the browser capabilities to its limits, it was extremely difficult to make it work and add so much functionality to it. We strongly advice to use the same thumbnail and image settings and sizes that we've used for the best possible performance, of course you can tweek it and add your own thumnails width and height based on your poroject, to do this inside the IGP constructor set the thumbnailMaxWidth:489 and thumbnailMaxHeight:321, these parameters represent the maximum initial thumbnails width and height when the thumbnails are first showed and the scale is 1 (no scale).

All images that will be loaded in the grid MUST have the same width and height so please make sure that you save your images properly!

To have the best possible visual grid and use your own custom size images they should be saved as follow, first set the thumbnailMaxWidth and thumbnailMaxHeight parameters to your desired thumbnails size, based on this the thumbnails images must be saved as follows, width = thumbnailMaxWidth * 1.4 and height = thumbnailMaxHeight * 1.4 (this parameters represents the maximum initial thumbnails width and height when the thumbnails are first showed and the scale is 1 (no scale).

C) Infinite Grid Pro installation - top

IGP has two display types, reponsive and afterParent.

Responsive installation

In the download files there is a start folder and in this folder there are example html files for each display type. You can use one of them to copy and paste the required html and js code based on the skin and display type that you need. In this tutorial we have used the responsive.html file.

  • Copy and paste the content folder, java folder and media folder into the same folder with your html file, they are self explanatory.
  • Open responsive.html with a text editor as reference.
  • The javascript and CSS files must be imported in the head section of your html.
    •
  • A div is required into which IGP will be added as a child, so create a div and set an id for it, the id is important because it is passed in the IGP constructor, make sure it is unique. The IGP is responsive as follows, the width and height will adapt based on the maxWidth and maxHeight properties specified in the constructor, if the page is resized and the parent div width is smaller then the maxWidth property IGP will adapt it's size accordingly. If autoScale property is set to yes the grid will modify it's height to keep a correct aspect ratio.
    •
  • Next step is to set up the categories / galleries, to do this copy and paste the div element with the id myGallery into the body element, the id is passed to the IGP parameter galleryId:myGallery. For more info about how to create categories / galleries please read Setup galleries.

  • Next step is to initialize IGP with javascript, in the head section of your html page add the code from below. Please note that all parameters are described in the Constructor parameters section.

This is how IGP is installed in a HTML page, please read the Constructor parameters section, it will explain all settings in details.

Atter parent installation

This display type is resizing IGP width and height to fill the available parent div width and height, this is really useful if you have a responsive div and you want IGP to resize based on it's size, basially is like setting the width to 100% and height to 100%, the grid will adapt based on the parent width and height. To use this display type follow the same installation steps from Responsive installation section, the only difference is the displayType property which must be set to afterParent like this displayType:"afterParent". In the start folder there is a HTML file after-parent.html, you can use them as reference.

D) Constructor parameters - top

Please open responsive.html with a text editor as reference. These parameters represents the possible setting of IGP, they are all described below.

MAIN SETTINGS.

  • rightClickContextMenu:"default" - this can be developer, default or disabled. We would appreciate it if you can leave this feature set to developer.
  • instanceName:"myIGP0" - represents the IGP instance name, this is used to access API methods, it must be unique and add if you use multiple instances in the same page add a different number at the end for example myIGP1, myIGP2, myIGP2.
  • parentId:"myDiv" - represents the div into which IGP will be added.
  • mainFolderPath:"content" - The main folder path. This folder contains some important files like the skin, vector font, CSS file, etc
  • galleryId:"myGallery" - The IGP galleries div id, for more info read Setup galleries.
  • displayType:"responsive" - this can be responsive or afterParent.
  • dragDirection:"both" - the grid drag direction, it can both, horizontal, vertical or none
  • initializeOnlyWhenVisible:"no" - this can be yes or no, if set to yes it will initialize the grid only when visible on page, otherwise the grid will initialize on page start.
    • lTransitionType:"opacity" - represents the thumbnail image transition type when it first appears, it can opacity or motion.
  • autoScale:"yes" - this can be yes or no. If set to yes the IGP height will always be proportional to the IGP width, if set to no the height will be fixed based on the maxHeight property.
  • paralax:"yes" - this can be yes or no.
  • autoScroll:"yes" - this can be yes or no. If set to yes the IGP will auto scroll in the direction set in the autoScrollDirection.
  • autoScrollDirection:"left" - represents the grid autoscroll direction, it can down, up, left or right.
  • autoScrollSpeed:2 - an integer that represents the autoscroll speed in px.
  • enableVisitedThumbnails:"no" - this can be yes or no. If set to yes the the thumbnails visited will have a reduced opacity this way the user can see which thumbnails content was viewed.
  • visitedThumbnailsOverlayOpacity:.2 - The thumnails overlay opacity, a number from 0 to 1 where 0.5 is 50% opacity.
  • showLookScrollButton:"yes" - this can be yes or no, If the mouse wheel or fingers are used on desktop to zoom in or out the grid this icon can disable the default grid zoom and drag functionality on mobile so that the user can scroll the page.
  • keepThumbnailsOriginalSizeOnGridStart:"no" - this can be yes or no. If set to yes the the when the grid is initialized the thumbnails will have the width equal to maxWidth and height equal to maxHeight. If set to no when the grid is initialized the thumbnails will be resized to fill the entire grid without cutting the thumbnails, this means that they will have a smaller or equal size then maxWidth and maxHeight.
  • addZoomSupport:"yes" - this can be yes or no. Enable or disable zoom on desktop and pinch & zoom on mobile devices.
  • addDragAndSwipeSupport:"yes" - this can be yes or no. Enable or disable drag and swipe.
  • disableThumbnailInteractivity:"no" - this can be yes or no. Disable or enable thumbnails interactivity, if is set to yes the thumbnail can't be clicked or touched to open the lightbox or a link.
  • showThumbnailOverlay:"yes" - this can be yes or no. Show or hide the thumbnail overlay when a thumbnail is hovered.
  • showThumbnailIcon:"yes" - this can be yes or no. Show or hide the thumbnail small media icon when a thumbnail is hovered.
  • openOnSingleTouch:"no" - this can be yes or no. On mobile open the lightbox on a single touch.
  • showCaption:"yes" - this can be yes or no. Show or hide thumbnail caption.
  • showCaptionOnThumbVideo:"yes" - this can be yes or no. Show or hide thumbnail caption if it has video.
  • maxWidth:1920 - maximum width in pixels, think of this property as it would be the max-width css property (only applies if the displayType is responsive).
  • maxHeight:900 - maximum height in pixels, think of this property as it would be the max-height css property (only applies if the displayType is responsive).
  • offsetWidth:700 - start point in px when the grid will start to resize if autoscale is set to yes, If autoscale is set to no this parameter will be ignored.
  • minHeight:400 - minimum grid height in px when resized on small screens.
  • thumbnailMaxWidth:489 - a number that represents the thumbnails maximum width in px, when IGP initializes the first time it's sets a start thumbnail width based on this parameter, the thumbnail initial width will never be larger then the parameter value.
  • thumbnailMaxHeight:321 - a number that represents the thumbnails maximum height in px, when IGP initializes the first time it's sets a start thumbnail height based on this parameter, the thumbnail initial height will never be larger then the parameter value.
  • maxScale:1.6 - the maximum allowed scale when the grid is zoomed, maximum allowed is 4.
  • minScale:.8 - the minimum allowed scale when the grid is zoomed, the minimum allowed scale is .6.
  • thumbnailOverlayOpacity:.7 - a number from 0 to 1 where 0.5 is 50% opacity.
  • backgroundColor:#000000 - the main background color.
  • thumbnailBackgroundColor:#111111 - the thumbnails background color.
  • thumbnailOverlayColor:#000000 - the thumbnails overlay color.

Add Lightbox video or audio:

To add a lightbox video or audio please check the markup below, the data-rl-src is required, it is the lightbox source, in this case the video or audio src, supported formats are:

  • .mp4 - add the video .mp4 path as the source, ex: data-rl-src="media/videos/fwd-1080p.mp4".
  • .mp3 - add the video .mp3 path as the source, ex: data-rl-src="media/audio/audio1.mp3".
  • Youtube - add the Youtube video URL as the source, ex: data-rl-src="https://www.youtube.com/watch?v=IDZTCLNOfqU"
  • Vimeo - add the Vimeo video URL as the source, ex: data-rl-src="https://vimeo.com/307791576".
  • HLS - add the HLS video URL as the source, ex: data-rl-src="https://bitdash-a.akamaihd.net/content/MI201109210084_1/m3u8s/f08e80da-bf1d-4e3d-8899-f0f6155f6efa.m3u8"
  • Google Drive - add the Google Drive video URL as the source, ex: data-rl-src="ttps://www.googleapis.com/drive/v3/files/1CaDKmLq_Ta_IEPUq7XK7oy7uTppQ2uLj?alt=media&key=AIzaSyCpMiFIS8G-1XrZmYXe9crmqhRM7sWC5js&v=.mp4".
  • Google Maps - add the Google Maps URL as the source, ex: data-rl-src="https://www.google.com/maps/embed/v1/place?q=castelul%20hunedoara&key=AIzaSyDYlgLIneg_UOd8STBfJEgq2JgmT5nNJKU".
  • IFRAME - add the the link of any page as the source, ex: data-rl-src="https://your-page.com".
  • PDF - add the .pdf as the source, ex: data-rl-src="media/sample.pdf".

The data-thumb-caption is optional and represents the thumbnail caption.

The data-rl-caption is optional and represents the lightbox caption.

G) API specifications - top

Inside the start file there is HTML file called API-example.html, open it with a text editor as reference. Below is a table with all API events and methods.

Methods:

Name Method Description

updateCategory

 updateCategory(categoryId:int):void Update the category based on id.

getCategoryName

 getCategoryName(categoryId:int):String Returns the current category name, it has an optional parameter that allows to get the category name based on id, for example if you need the second category name this method can be called like this myIGP.getCategoryName(1);.

getCategoryId

 getCategoryId():Number Returns the current category id, starting from 0 (zero).

Event listeners:

Event Prefix Description
ready FWDIGP.READY Dispatched when IGP API is ready to be used.
categoryUpdate FWDIGP.CATEGORY_UPDATE Dispatched when IGP a category is changed.
thumbVideoStart FWDIGP.THUMB_VIDEO_START Dispatched when the thumbnail video has started.
thumbVideoPlay FWDIGP.THUMB_VIDEO_PLAY Dispatched when the thumbnail video playes.
thumbVideoPause FWDIGP.THUMB_VIDEO_PAUSE Dispatched when the thumbnail video is paused.
thumbVideoPlayComplete FWDIGP.THUMB_VIDEO_PLAY_COMPLETE Dispatched when the thumbnail video has finished playing.
goFullScreen FWDIGP.GO_FULLSCREEN Dispatched when the thumbnail video is going fullscreen.
goNormalScreen FWDIGP.GO_NORMALSCREEN Dispatched when the thumbnail video is going normalscreen.
error FWDIGP.ERROR Dispatched when an error occurs, it has a parameter named error which returns the error text.
showStart FWDIGP.LIGHTBOX_SHOW_START Dispatched when IGP lightbox starts to show.
showComplete FWDIGP.LIGHTBOX_SHOW_COMPLETE Dispatched when IGP show animation is complete.
hideStart FWDIGP.LIGHTBOX_HIDE_START Dispatched when IGP lightbox starts to hide.
hideComplete FWDIGP.LIGHTBOX_HIDE_COMPLETE Dispatched when IGP hide animation is complete.
